# EZWorks Expense Project Instructions

## Documentation Discipline
- 주요 정책, UX 결정, 운영 주의사항, 템플릿/환율/문서정책 관련 변경은 `docs/product/expense_processor_major_notes.md`에 계속 누적 기록한다.
- 기록 대상은 "나중에 다시 봤을 때 왜 이렇게 구현했는지"를 이해하는 데 도움이 되는 사항으로 제한한다.
- 단순 UI 문구 수정이나 사소한 스타일 조정처럼 회고 가치가 낮은 변경은 기록하지 않아도 된다.
- 새로운 구조 변경이 들어가면 관련 파일 경로도 함께 남긴다.

## Commenting Standard
- 핵심 로직, 정책 분기, 상태 동기화, 템플릿 선택, FX 계산, 권한/테넌트 분기 같은 주요 코드에는 한국어 주석을 추가한다.
- 주석은 "무슨 코드인지"보다 "왜 이 흐름이 필요한지", "어떤 상태를 맞추는지"를 설명하는 데 집중한다.
- 모든 줄에 주석을 달지 말고, 다음 사람이 다시 읽을 때 해석 비용이 높은 블록과 함수 위주로 남긴다.
- 새로 추가하는 복잡한 함수에는 가능하면 함수 상단에 한 줄 이상의 한국어 설명을 둔다.

## Implementation Habit
- 기능을 수정할 때는 코드 반영만 하지 말고, 위 두 원칙을 함께 확인한다.
- 큰 기능 흐름을 손댔으면:
  1. 필요한 한국어 주석을 추가하고
  2. `docs/product/expense_processor_major_notes.md`에 핵심 결정사항을 기록한다.
